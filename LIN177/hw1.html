HW 1: Wreck a nice beach

Part 1: Pronunciation FST

The CMU pronunciation dictionary is a resource widely used for speech recognition (speech to text) and speech synthesis (text to speech). It is a large text file containing one word (in English) per line, along with a transcription of its pronunciation. It is in some ways similar to the word pronunciation information included with standard dictionaries.

Start by going to the CMU pronunciation dictionary webpage. Read the basic information contained there, but don't worry about the download and stress information; focus mainly on the Phoneme Set section, and try to see how plain text characters are used to represent how a word is pronounced. For example, the symbol G is used to represent the g sound in "green", the symbol AA is used to represent the sound the letter o makes in "odd", and the symbol EY is used to represent the sound the letter a makes in "ate". Don't worry about remembering each of the symbols and how they sound. Just get a general sense for how the word pronunciations are represented.

Now look at the  actual dictionary file . 

(The dictionary starts with pronunciation for punctuation and other symbols. Scroll down to see pronunciation for words. The words are in alphabetical order.)

Examine the dictionary entries for the following four words: language, languages, student, and students. Draw an FST that will use pronunciation representations as input, one symbol at a time, and produce the corresponding words as output. The input alphabet should be the set of phonemes in the CMU dictionary page (i.e. Σ = {AA, AE, AH, AO, AW, B, CH, D, DH, ...}), and the output alphabet should be exactly Γ = {language, languages, student, students}. The FST should map the pronunciations to their corresponding words. (Pay close attention: the output alphabet is not the set of letters; it is those four symbols in the set Γ.) The output language should be Γ*.

For example, if the input string is the sequence of symbols

L AE NG G W AH JH AH Z

the output string should be

languages 

(which is, in this case, a string made of just one symbol of the output alphabet).

If the input string is the sequence of symbols

L AE NG G W AH JH S T UW D AH N T S

the output string should be

language students 

 

Part 2: Prolog FST

Consider the following FST, represented as Prolog knowledge statements . You don't need to know Prolog to understand this way to represent FSTs; this is exactly the notation seen in class. This is just an example, but it could be helpful to draw what this FST looks like. (This is not required, but recommended if you can't see exactly what the transducer does when looking at the Prolog knowledge statements.)

transition(1, iy, 2, eps).
transition(2, z, 3, eps).
transition(3, iy, 1, easy).
transition(1, iy, 4, eps).
transition(4, z, 5, eps).
transition(5, iy, 6, eps).
transition(6, g, 7, eps).
transition(7, ow, 8, eps).
transition(8, ih, 9, eps).
transition(9, ng, 1, easygoing).
initial(1).
final(1).

Each line represents a transition from one state to the other. The first argument is the state where the transition starts, the second argument is the input symbol for the transition, the third argument is the state where the transition goes, and the last argument is the output symbol. Epsilon is represented as eps, and the pronunciation symbols are lowercased, so that they are not interpreted as prolog variables (we will discuss this in class, but make sure to use lowercase for the pronunciation symbols). The initial state is state 1, and the only final state is also state 1.

Your task in part 2 is to write the FST you designed in part 1 in the prolog format illustrated above (i.e. using the transition/3, initial/1 and final/1 predicates).

(Hint: if the answers to parts 1 and 2 don't seem clear initially, try examining the example above, and draw the FST diagram that corresponds to the prolog statements provided.)

You can try using your prolog FST here:

http://compling.ucdavis.edu/taufst

Enter your FST where it says "Enter FST specification." Then, where it says "Enter a query", you can use your FST by typing

fst([l, ae, ng, g, w, ah, jh, ah, z], [languages]).

The query uses the predicate fst/2 (the /2 means that fst takes two arguments). The first argument is the input string, represented as a list of symbols. The list is enclosed in squre brackets, and the symbols in the string are separated by commas. The second argument is the output string, which in this case is a list of symbols containing just one symbol (languages). When we issue this query, prolog should tell us that this is true. In other words, our FST maps the input string [l, ae, ng, g, w, ah, jh, ah, z] to the output string [languages].

Instead of specifying an output string, we can instead use a variable. In prolog, variables start with uppercase letters. For example, if we try this query:

fst([l, ae, ng, g, w, ah, jh, ah, z], W).

prolog will try to find a value for W that makes the query true. If it finds such a value, it tells us what it is. This also works if we specify the output and use a variable as the input.

fst(X, [languages]).

If there are multiple values of X that make the query true, we will first get one value from prolog, but the value will be followed by a semi-colon. If we then press enter (or return), prolog will search for another value that works. If it finds one, it will show it to us. If it doesn't find other values that work, it will say "false".

 

Part 3: Beach wrecking

In this last part of the assignment you will download a file called cmu_dictionary.pl from canvas (look in Files, in the HW1 folder, or just use the link to download). Once you have the file, load it into the FST app. The file contains a larger FST corresponding to a fragment of the CMU pronunciation dictionary (not all the words in the dictionary are included).

Try the following:

fst(T, [against]).

That should give you T = [ah, g, eh, n, s, t]. You used the transducer to find the pronunciation of the word student. Notice that after [ah, g, eh, n, s, t], there is a semi-colon. This indicates that there may be other values of T that work. Press enter (or return) to get another value (after a few seconds): [ah, g, ey, n, s, t]. These are two different ways to pronounce against, according to the CMU pronunciation dictionary. Again, the semi-colon appears after the answer. Pressing enter again will cause prolog to look for another answer for several seconds. In this case, it doesn't find one, and says false.

Now try

fst([ah, g, eh, n, s, t], W).

You should get W = [against]. This time, you used the transducer to find which word has the given pronunciation.

Now try

fst(T, [ice, cream]).

Note the value of T, which is a representation of the pronunciation of ice cream. Use this pronunciation as the input:

fst(enter the input here, W).

(Don't type "enter the input here"; you need to get the actual list from the previous query.)

The FST will give you the words that correspond to the pronunciation provided.

Once you get a result, press the enter (or return) key. That will give you additional results, if there are any. Keep pressing enter until you reach the end, when it says "false" (getting "false" after several valid strings doesn't means it failed in general, it just means that you ran out of results after succeeding several times).

What values of W did you get?

Finally, think about whether there are any sequences of words in English that sound like "finite state". Use the same idea as in the "ice cream" example above to see if our pronunciation FST can help us find sequences of English words that sound like "finite state". (You will need to use the string [finite, state], instead of the string [ice, cream]).

First, find the string of sound symbols that represents the sounds corresponding to [finite, state]. Then, find the output string(s) that correspond to that string of sound symbols.

What happens? Did you find sequences of English words that sound like finite state? List what you typed in prolog to solve this problem, and describe the result. Why was this the result?

(And that's only part of why it's hard to wreck a nice beach!)
