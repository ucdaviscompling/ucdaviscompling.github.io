HW 5: Don't arrive the dog

In this assignment, we will again write grammars using SWI Prolog online:

https://swish.swi-prolog.org

 

Part 1: Arrived the dog

Consider the CFG below, implemented in prolog:

s --> np, vp.
np --> det, n.
np --> pro.
vp --> v.
vp --> v, np.

det --> [the].
n --> [dog].
n --> [dogs].
pro --> [she].
pro --> [her].
pro --> [they].
pro --> [them].
pro --> [i].
pro --> [we].
v --> [arrived].
v --> [like].
v --> [likes].

This grammar accepts simple sentences like

she likes the dog

and 

the dog arrived

It also accepts sentences like

her likes them

and

they arrived the dog

In fact, we can check all of the sentences in the language defined by this grammar using the following query:

s(S, []).

S = [the, dog, arrived] 
S = [the, dog, like] 
S = [the, dog, likes] 
S = [the, dog, arrived, the, dog] 
S = [the, dog, arrived, she] 
...

And the list goes on.

In part 1 you will model a specific version of English by refining this grammar with:

  • Case information, so that I, we, she and they can be used as subject but not as objects, and her and them can be used as objects, but not as subjects). This should rule out Her arrived, while allowing She arrived.
  • Subject-verb agreement, so that present tense verbs with third-person singular subjects are inflected with the -s suffix (e.g. like + -s -> likes). This will involve adding person and number information to verbs and nominals. This should rule out She like the dog, while allowing She likes the dog, and They see the dog.
  • Verb subcategorization, to differentiate between transitive and intransitive verbs. This should rule out sentences like She arrived them.

As a starting point, consider the plain grammar above, which does not account for case, subject-verb agreement, or verb subcategorization. Now, consider the grammar below, which has been modified to account for case. Notice that the grammar now differentiates between they and them, so they like they is no longer a string in the language defined by the grammar.

% the sentence rule
% the subject should have nominative case
% (denoted in the grammar as np_subj)
s --> np_subj, vp.

% noun phrase rules
np_subj --> det, n.
np_subj --> pro_subj.
np_obj --> det, n.
np_obj --> pro_obj.

% verb phrase with intransitive verb
vp --> v.

% verb phrase with transitive verb
% the object should have accusative case
% (denoted in the grammar as np_obj)
vp --> v, np_obj.

% lexical rules
det --> [the].
n --> [dog].
n --> [dogs].
pro_subj --> [she].
pro_obj --> [her].
pro_subj --> [they].
pro_obj --> [them].
pro_subj --> [i].
pro_subj --> [we].
v --> [arrived].
v --> [like].
v --> [likes].

Inspect the grammar above carefully, comparing it to the plain grammar. What changed?

  • In the lexical rules, the pronouns that all had the category pro were split into two categories: pro_subj (pronouns with nominative case, which can be used as subjects), and pro_obj (pronouns with accusative case, which can be used as objects).
  • The noun phrase rules that used pronouns (with category pro) were changed to take case into account, by using either pro_subj or pro_obj.
  • The non-terminal NP was replaced with two new non-terminals: NP_subj and NP_obj. The NP rules were changed accordingly: with pronouns, the case is simply passed on to the noun phrase, and with determiner noun, the noun phrase can have either case (in English).
  • Now that our NPs have case, the VP rule for transitive verbs was changed, so that the object of a verb must have accusative case (i.e. only NP_obj can be an object, not NP_subj).
  • Similarly, the S rule was changed to that only NP_subj can be a subject.

Your task: now that case has been taken care of, modify the grammar further to account for subject-verb agreement and verb transitivity. Name your modified grammar hw5part1.pl.

Your modifications will likely involve changes to the lexical rules (splitting existing word categories into multiple categories, as we did with pro to account for case) and changing the phrase structure rules to use the new word categories.

Your grammar should only generate English sentences that observe case restrictions for subject and object, subject-verb agreement, and transitivity as described above. Additionally, the grammar should be principled. For example, the grammar must reflect that the verb likes is used with the subject she is that there is person and number agreement between the subject and the verb (namely, the subject is third-person singular, and the verb is third-person singular), and not just that the symbol likes happens to go with the symbol she. The goal is not simply to produce the correct set of sentences, but to create a grammar that is accurate, simple and principled. Using too many rules, too many non-terminals, or rules or non-terminals that don't reflect how agreement or transitivity work in English will not result in the desired grammar.

Hint: If we think of what hw5part1plain.pl does, there are three different things to work on: case, subject-verb agreement, and verb subcategorization. The first one is done: hw5part1case.pl. If all of the changes together seem too confusing to keep track of, do agreement and transitivity, one at a time. These are separate issues. Then put it all together.

What to turn in for part 1: hw5part1.pl (your modified grammar).

 

Part 2: The features arrived the grammar

Splitting categories and non-terminals gets out of hand fast. An alternative to the proliferation of non-terminals is to attach features to the terminals and non-terminals, and use these features to constrain the grammar. 

Consider the case example in part 1. We split pro into pro_subj and pro_obj, and we split np into np_subj and np_obj. The sequence det n could be either, so we added two rules (one for np_subj, and one for np_obj) with identical right-hand sides. Instead of splitting the symbols of the grammar, we can modify the lexical entries as follows:

pro --> [she].
pro --> [her].

could become

pro(subj) --> [she].
pro(obj) --> [her].

So far this is not so different from splitting pro into pro_subj and pro_obj. However, the difference is more apparent when we modify the NP grammar rules:

np_subj --> pro_subj.
np_obj --> pro_obj.
np_subj --> det, n.
np_obj --> det, n.

now become

np(C) --> pro(C).
np(_) --> det, n.

In the first rule, the NP inherits its case feature C from the pronoun, and the pronoun gets it from the lexical rule. Recall that we already changed the lexical rules, so for the word she, we would have pro(subj), which can make an np(subj). In the second rule, instead of using an ordinary prolog variable, we used the underscore, which is called the anonymous variable; it matches anything, so we are just leaving the case feature unspecified (it can be set to something else in another rule).

In this simple example we see that instead of expanding two rules into four, we keep the number of rules intact, but still constrain the grammar in the same way.

The case information can be used in a VP rule as below:

vp --> v, np(obj).

Compare this rule to the corresponding rule in the grammar that accounts for case in part 1. They do the same thing. Now look at the grammar below, which is similar to the grammar in part 1, but implements case using features instead.

% sentence
% the subject NP must have nominative case
s --> np(subj), vp.

% noun phrase
% the NP gets case information from its head
% (either the pronoun or the noun)
np(F) --> pro(F).
np(F) --> det, n(F).

% verb phrase with intransitive verb
vp --> v.

% verb phrase with transitive verb
% the object NP must have accusative case
vp --> v, np(obj).

% lexical rules
det --> [the].
% the underscores below can match anything,
% so these nouns could be used in nominative
% or accusative NPs
n(_) --> [dog].
n(_) --> [dogs].
pro(subj) --> [she].
pro(obj) --> [her].
pro(subj) --> [they].
pro(obj) --> [them].
pro(subj) --> [i].
pro(subj) --> [we].
v --> [arrived].
v --> [like].
v --> [likes].

Examine this grammar carefully, comparing it to grammar in part 1. Notice how instead of creating more non-terminals, the grammar uses features. The final step is to implement a grammar that accounts for case, subject-verb agreement and verb subcategorization, but this time using features.

Use the following as a starting point. Notice that it includes extra features, which should be useful for implementing subject-verb agreement and verb subcategorization.

% sentence
% we need to enforce subject-verb agreement
% (person and number need to match for NP and VP),
% and we need the right case
s --> np(subj, ADD_MORE_FEATURES), vp(ADD_FEATURES_HERE).

% noun phrase
% ADD THE NP RULES

% verb phrase
% VP with intransitive verb
% the VP gets Person and Number from the V
vp(Person, Number) --> v(intransitive, Person,Number).

% VP with transitive verb
% ADD THE TRANSITIVE VP RULE

% lexical rules

det --> [the].

% pronoun, subjective case, third person singular
pro(subj, third, singular) --> [she].

% WRITE THE LEXICAL ENTRIES FOR:
% her, they, them, i, we

% here the underscore means that case can be
% anything. (dog is the same, whether it is
% subj or obj.)
n(_, third, singular) --> [dog].

% WRITE THE LEXICAL ENTRY FOR dogs

% verb features:
% - transitive/instransitive
% - person: first, second, third
% - number: plural, singular

% the underscores here mean that Person can
% be anything, and Number can be anything
v(intransitive, _, _) --> [arrived].

% here we need to ensure person/number is
% third singular
% WRITE THE LEXICAL ENTRY FOR likes (transitive)

% person/number can be anything (first/second/third, singular/plural), except for
% third singular
v(transitive, first, _) --> [like].
v(transitive, second, _) --> [like].
v(transitive, third, plural) --> [like].

Your task: starting from hw5part2template.pl, implement case, subject-verb agreement and verb subcategorization using features.

When you are done, the query

s(S, []).

should give you only grammatical sentences as values of S.

Again, your grammar should be principled, accurate and simple. It is trivial to implement a grammar that produces the correct sentences somehow, but this is not sufficient. Your grammar should implement subject-verb agreement and verb subcategorization using features in a way that does not result in an unnecessary increase in the number of grammar rules, and, more importantly, model how transitivity and subject-verb agreement work in English.

What to turn it: your grammar, in a file called hw5part2.pl.
